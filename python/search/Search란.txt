순차 탐색이란?
    리스트 안에 있는 특정한 데이터를 찾기 위해 앞에서부터 데이터를 하나씩 차례대로 확인하는 방법
    보통 정렬되지 않은 리스트에서 데이터를 찾을 때 사용
    데이터가 아무리 많아도 시간만 충분하다면 항상 원하는 원소를 찾아낼 수 있음
    count()함수가 실행되면 순차탐색이 진행됨
    시간복잡도 : 최악의 경우 O(N)

이진 탐색이란?
    데이터가 정렬되어 있어야 사용 가능한 알고리즘으로, 정렬되어 있다면 매우 빠르게 데이터를 찾아 낼 수 있음
    3개의 변수 시작점, 끝점, 중간점을 이용
    찾으려는 데이터와 중간점에 위치한 데이터를 반복적으로 비교하여 원하는 데이터를 찾아냄
    시간복잡도 : O(logN)

데이터 탐색 범위가 2000만을 넘어간다면 또 데이터의 개수나 값이 1000만 단위를 넘어간다면 이진 탐색과 같은 O(logN)의 속도를 내야 하는
알고리즘을 떠올려야 한다

입력받을 데이터의 개수가 1000만개 이상이거나 범위가 1000억 이상이라면 이진 탐색 알고리즘을 의심해야함
input()은 입력받는 속도가 느려 시간초과가 날 수 있으므로, 이럴 경우에는
sys.readline().rstrip()을 이용
